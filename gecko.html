Project Overview

Gecko is a complete compiler for a custom programming language, built from scratch in Java. The project covers all the fundamental phases of the compilation process, starting with a lexical analysis to tokenize the source code, followed by a recursive descent syntactic analysis to validate the code's structure against a defined context-free grammar. Subsequently, a semantic analysis is performed to verify the program's coherence, managing rules for typing, variable scope, and declarations. Finally, the compiler constructs an Abstract Syntax Tree (AST), which it uses to generate intermediate code in a three-address format, laying the groundwork for subsequent machine code generation. The project also stands out for implementing its own data structures, such as lists and stacks, to manage the compiler's internal components.

Key Features

Lexical Analysis and Tokenization

The compiler implements a lexical analyzer that processes the input source code to convert it into a sequence of tokens. It identifies and classifies lexemes such as keywords, identifiers, numbers, and operators, generating a token table and a symbol table as output.

Recursive Descent Syntactic Analysis

It uses a parser based on the recursive descent method to verify that the token sequence conforms to the context-free grammar defined for the Gecko language. It validates the correct structure of statements, expressions, and code blocks.

Performance Optimization

It incorporates a semantic analyzer that applies critical rules for program logic. It manages variable scope using a stack of symbol tables and performs type compatibility checks to ensure that operations are valid.

Abstract Syntax Tree (AST) Construction

During the analysis, the compiler builds an Abstract Syntax Tree (AST) that hierarchically represents the code's structure. This tree is a key component for subsequent phases, as it captures the program's syntactic essence in an organized manner.

Intermediate Code Generation

From the AST, the compiler traverses the tree to generate three-address code. This is an intermediate representation of the source code that simplifies optimization and the final translation into machine or assembly code.

Custom Data Structures

For complete control over data management, the project was developed by implementing its own data structures from scratch, including linked lists (Lista), stacks (Pila), and tree nodes (nodoArbol), without relying on Java's standard libraries.
